## Quickstart — 1 command

```bash
# Make the script executable (first time only)
chmod +x start.sh

# Set up + launch both servers
./start.sh
```

This installs all dependencies, auto-generates a secure `SESSION_SECRET` in `backend/.env`, and starts both servers.

- **Backend** → http://localhost:4000
- **Frontend** → http://localhost:5173

## Authentication Flow

```
Browser                 Backend                    SRM Academia
  │                        │                             │
  │  POST /api/auth/login  │                             │
  │  { username, password }│                             │
  │ ──────────────────────►│                             │
  │                        │  1. verifyUser(username)    │
  │                        │ ───────────────────────────►│
  │                        │ ◄── { identifier, digest }  │
  │                        │                             │
  │                        │  2. verifyPassword(...)     │
  │                        │ ───────────────────────────►│
  │                        │ ◄── { cookies, isAuth }     │
  │                        │                             │
  │                        │  Store cookies in session   │
  │                        │  (never sent to browser)    │
  │ ◄── Set-Cookie: sid    │                             │
  │ ◄── { userInfo }       │                             │
  │                        │                             │
  │  GET /api/sync  ───────►                             │
  │                        │  getAttendance(cookies)     │
  │                        │  getMarks(cookies)      ───►│
  │                        │  getTimetable(cookies)      │
  │ ◄── { data }           │                             │
```

**Security:**
- SRM password is **never stored** anywhere
- SRM session cookies live **only on the backend** inside `express-session`
- Browser gets only a httpOnly session cookie (`knowit.sid`)
- Login rate-limited: 10 requests / 15 minutes / IP
- Session TTL: 6 hours

## Frontend Data Flow

```
App.jsx
 └─ AuthProvider
      └─ useStudentData(loggedIn)
            │
            ├─ Hydrate from localStorage cache   → instant render
            ├─ GET /api/sync                     → live data
            ├─ On success: update state + cache  → green dot
            └─ On failure: keep mock/cache       → amber dot
```

All pages receive data as props:
```
{ attendance, marks, timetable, isSyncing, lastSynced, syncError, isLive, onSync }
```

# Set environment variables on your host:

CLIENT_ORIGIN=https://your-app.vercel.app

```bash
# Build
npm run build

# Environment variable:
VITE_API_URL=https://your-backend.railway.app
```

> **CORS:** `CLIENT_ORIGIN` on the backend must exactly match your frontend URL — no trailing slash, must include `https://`.

## Troubleshooting

**`srm-academia-api` login fails**
→ The SRM portal (`academia.srmist.edu.in`) may be down or have updated their API. Check the package's GitHub for patches.

**Session cookie not sent**
→ In dev, Vite's proxy handles this automatically. In prod, ensure `sameSite:"none"` + `secure:true` and both services are on HTTPS.

**Attendance / marks shows 0**
→ The response field names from `srm-academia-api` may have changed. Update the field mappings in `academia.service.js`.

**`NETWORK_ERROR` in sync bar**
→ Backend isn't running. Start it with `npm run dev` in `/backend`.

**401 after some time**
→ Session expired (6h). The sync bar shows the error — user logs in again.
